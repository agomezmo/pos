2. Arquitectura

El sistema consta de tres servicios que se ejecutan como contenedores Docker:

┌─────────────────────────────────────────────────────────────┐
│                    DOCKER NETWORK                            │
│                                                             │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐   │
│  │   Frontend   │    │  API .NET    │    │  PostgreSQL  │   │
│  │   (nginx)    │───▶│  :5001       │───▶│  :5433       │   │
│  │   :8081      │    │              │    │              │   │
│  └──────────────┘    └──────────────┘    └──────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
  • Frontend: React + Vite servido por nginx en :8081
  • API: .NET 8 Web API en :5001
  • Base de datos: PostgreSQL 15 en :5433

3. Instalación con Docker (Recomendada)

3.1 Clonar el Repositorio

git clone https://github.com/agomezmo/pos.git
cd pos

3.2 Configurar Variables de Entorno

Edita el archivo app.env (o docker-compose.yml) con tus valores:

# Base de datos
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_DB=pos_farmacia

# JWT
JWT_SECRET=tu_clave_secreta_segura

3.3 Iniciar los Servicios

docker compose up -d

Esto iniciará los tres contenedores (db, api, frontend).

Verificar estado:
docker compose ps
docker compose logs -f api

3.4 Verificar la Instalación

  • Frontend: http://localhost:8081
  • API (swagger): http://localhost:5001/swagger
  • API health: curl http://localhost:5001/health
Instalación completada. El sistema está listo para usar.

4. Instalación Manual

4.1 Base de Datos PostgreSQL

sudo apt install postgresql postgresql-contrib -y
sudo systemctl start postgresql
sudo systemctl enable postgresql
sudo -u postgres psql -c "CREATE DATABASE pos_farmacia;"
sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'postgres';"
psql -h localhost -U postgres -d pos_farmacia -f database/schema.sql
psql -h localhost -U postgres -d pos_farmacia -f database/seed_pharmacy.sql

4.2 API .NET

# Requiere .NET SDK 8.0
cd backend-net
dotnet restore
dotnet build --configuration Release
dotnet publish --configuration Release --output /opt/pos/api

# Configurar como servicio systemd
sudo tee /etc/systemd/system/pos-api.service << 'EOF'
[Unit]
Description=POS Farmacia API
After=network.target postgresql.service

[Service]
WorkingDirectory=/opt/pos/api
ExecStart=/usr/bin/dotnet /opt/pos/api/POS.API.dll
Restart=always
RestartSec=10
Environment=ASPNETCORE_URLS=http://0.0.0.0:5001

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now pos-api.service

4.3 Frontend

cd frontend
npm install
npm run build
sudo cp -r dist/* /var/www/pos/

5. Compilación del Frontend

Para compilar el frontend React manualmente:

cd frontend
npm install
npm run build

Esto genera los archivos estáticos en frontend/dist/.

Para desarrollo con hot-reload:

npm run dev

El servidor de desarrollo corre en http://localhost:5173 y hace proxy a la API en :5001.

6. Configuración

6.1 Variables de Entorno (API)

Archivo app.env:

# Base de datos
DB_HOST=pos_farmacia_db
DB_PORT=5432
DB_NAME=pos_farmacia
DB_USER=postgres
DB_PASSWORD=postgres

# JWT
JWT_SECRET=ClaveSuperSegura123!
JWT_ISSUER=POS.Farmacia
JWT_AUDIENCE=POS.Farmacia.App
JWT_EXPIRATION_HOURS=8

# Configuración
ASPNETCORE_ENVIRONMENT=Production
ASPNETCORE_URLS=http://0.0.0.0:5001

6.2 Nginx (Frontend)

El archivo nginx.conf/default.conf configura:

  • Servir archivos estáticos del frontend.
  • Proxy inverso de /api a la API en :5001.
  • Compresión gzip.
  • Headers de seguridad.

7. Base de Datos

7.1 Esquema

El esquema se encuentra en database/schema.sql e incluye:

  • Companies — Información de la empresa
  • Users — Usuarios del sistema
  • Categories — Categorías de productos
  • Products — Catálogo de productos
  • Customers — Clientes
  • Suppliers — Proveedores
  • Sales / SaleItems — Ventas y detalle
  • CashRegisterSessions — Sesiones de caja
  • Expenses — Gastos
  • Returns / ReturnItems — Devoluciones
  • Patients — Pacientes
  • Prescriptions / PrescriptionItems — Recetas
  • InventoryMovements — Movimientos de inventario
  • Alerts — Alertas del sistema
  • Facturas — Facturación CFDI
  • CompanySettings — Configuración general

7.2 Backup y Restauración

# Backup
docker exec pos_farmacia_db pg_dump -U postgres pos_farmacia > backup_$(date +%Y%m%d).sql

# Restore
cat backup.sql | docker exec -i pos_farmacia_db psql -U postgres pos_farmacia

9. Solución de Problemas

Error: "Connection refused" a la API

  • Verifica que el contenedor esté corriendo: docker compose ps
  • Revisa logs: docker compose logs api
  • Asegúrate de que el puerto 5001 no esté ocupado: sudo lsof -i :5001

Error: "Cannot connect to database"

  • Verifica que PostgreSQL esté listo: docker compose logs db
  • Revisa credenciales en app.env
  • Prueba conexión manual: docker compose exec db psql -U postgres -d pos_farmacia

Error: "401 Unauthorized"

  • El token JWT ha expirado. Vuelve a iniciar sesión.
  • Verifica la hora del servidor (debe estar sincronizada con NTP).
  • Revisa la clave JWT_SECRET en la configuración.

Error: Frontend en blanco o no carga

  • Verifica que nginx esté corriendo: docker compose logs frontend
  • Limpia caché del navegador (Ctrl + Shift + R).
  • Revisa la consola del navegador (F12) por errores.

Error: Docker no tiene permisos

sudo usermod -aG docker $USER
# Cerrar sesión y volver a iniciar

10. Actualización

Actualizar desde Git

git pull origin main
docker compose down
docker compose build --no-cache api
docker compose up -d

Migraciones de Base de Datos

Las migraciones se aplican automáticamente al iniciar la API mediante Entity Framework. Para aplicar migraciones manualmente:

docker compose exec api dotnet ef database update

Rollback

git revert HEAD
docker compose down
docker compose up -d
Nota: Siempre realiza un backup de la base de datos antes de actualizar.

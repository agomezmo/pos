Relaciones Clave

  • Ventas → Productos: Una venta (Sales) contiene uno o muchos productos a través de SaleItems (relación N:M con datos extra).
  • Caja → Ventas/Gastos: Cada sesión de caja (CashRegisterSessions) agrupa las ventas y gastos realizados durante un turno.
  • Productos → Inventario: Cada movimiento de inventario (InventoryMovements) registra cambios en el stock de un producto.
  • Recetas → Pacientes: Un paciente puede tener múltiples recetas a lo largo del tiempo.
  • Devoluciones → Ventas: Una devolución está vinculada a una venta original y revierte el inventario de los productos devueltos.
  • Facturas → Ventas: Una venta puede generar una o más facturas (según el esquema de facturación).

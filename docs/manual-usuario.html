1. Introducción

POS Farmacia es un sistema de punto de venta diseñado específicamente para farmacias. Permite gestionar ventas, inventario, clientes, proveedores, pacientes, recetas médicas, caja y facturación electrónica (CFDI).

El sistema está compuesto por una interfaz web moderna accesible desde cualquier navegador en la red local, una API REST en .NET y una base de datos PostgreSQL.

Acceso: Abre tu navegador y ve a http://localhost:8081 (o la IP del servidor).

2. Acceso al Sistema

Pantalla de Login

Al ingresar al sistema verás la pantalla de inicio de sesión:

  • Usuario: Tu nombre de usuario asignado por el administrador.
  • Contraseña: La contraseña correspondiente.
  • Presiona Ingresar o Enter para acceder.

3. Panel Principal (Dashboard)

El dashboard muestra un resumen ejecutivo del negocio:

  • Ventas del día: Total de ventas del día actual.
  • Productos registrados: Total de productos en el catálogo.
  • Clientes registrados: Total de clientes en la base de datos.
  • Alertas activas: Número de alertas pendientes (stock bajo, vencimientos, recetas expiradas).
  • Acceso rápido: Botones para ir directamente a las funciones más usadas.
  • Alertas recientes: Lista de las últimas alertas generadas.

4. Módulo de Ventas (POS)

Pantalla de Nueva Venta

El módulo de ventas es el corazón del sistema. Interfaz tipo POS con:

  • Buscador de productos: Escribe el nombre o código del producto para agregarlo.
  • Carrito de compras: Lista de productos agregados con cantidad, precio y subtotal.
  • Totalización automática: Subtotal, descuento, IVA y total se calculan en tiempo real.
  • Selección de cliente: Opcional — buscar cliente por nombre o teléfono.
  • Método de pago: Efectivo, Tarjeta, Transferencia, Crédito.
  • Monto recibido y cambio: Al seleccionar "Efectivo", ingresa el monto y el sistema calcula el cambio.
  • Finalizar venta: Botón para completar la transacción.

Proceso de Venta

  1. Busca productos por nombre o código en el campo de búsqueda.
  2. Selecciona el producto deseado — se agrega automáticamente al carrito.
  3. Ajusta la cantidad si es necesario con los botones +/-
  4. Opcionalmente selecciona un cliente y aplica descuento.
  5. Selecciona el método de pago.
  6. Si es efectivo, ingresa el monto recibido.
  7. Presiona Cobrar para finalizar la venta.
  8. El sistema genera un comprobante con número de folio único.

Historial de Ventas

Muestra todas las ventas realizadas con opciones para:

  • Filtrar por fecha (rango de fechas).
  • Buscar por folio o nombre de cliente.
  • Ver detalle completo de cada venta (productos, totales, método de pago).
  • Generar factura desde una venta existente.

5. Módulo de Productos

Gestión de Productos

Catálogo completo de productos con:

  • Código: Código único de producto (código de barras o interno).
  • Nombre: Nombre del producto.
  • Descripción: Descripción detallada.
  • Categoría: Clasificación del producto.
  • Precios: Precio de compra y precio de venta.
  • Stock: Cantidad actual en inventario y stock mínimo.
  • Unidad: Unidad de medida (pza, caja, blíster, etc.).
  • Receta: Indicador si requiere receta médica.
  • IVA: Si el producto está sujeto a impuesto.
  • Proveedor: Proveedor asociado.
  • Lote y Fecha de vencimiento: Control de lotes y caducidad.
  • Activo/Inactivo: Estado del producto en el catálogo.

Acciones

  • Agregar producto: Formulario completo para nuevo producto.
  • Editar: Modificar datos del producto.
  • Activar/Desactivar: Cambiar el estado del producto (sin eliminarlo).
  • Buscar: Filtro por nombre, código o categoría.
  • Stock bajo: Ver productos por debajo del stock mínimo.
  • Por vencer: Ver productos próximos a caducar.

6. Categorías

Administra las categorías de productos:

  • Lista de categorías con nombre y descripción.
  • Crear, editar y eliminar categorías.
  • Las categorías ayudan a organizar los productos (Analgésicos, Antibióticos, Vitaminas, Cuidado Personal, etc.).

7. Clientes

Gestión de la base de clientes:

  • Registro: Tipo y número de identificación, nombre completo, teléfono, email, dirección.
  • Buscar: Por nombre, teléfono o documento.
  • Editar: Actualizar información del cliente.
  • Historial: Ver compras realizadas por el cliente (desde detalle de cliente).

8. Proveedores

Administra los proveedores:

  • Nombre del proveedor, persona de contacto, teléfono, email, dirección.
  • Crear, editar y buscar proveedores.
  • Asociar productos a proveedores.

9. Inventario

Movimientos de Inventario

Control de entradas y salidas de productos:

  • Entrada: Registrar ingreso de mercancía (compra, ajuste, devolución de cliente).
  • Salida: Registrar salida de mercancía (venta, ajuste, merma, devolución a proveedor).
  • Historial: Bitácora completa de todos los movimientos con fecha, usuario, tipo y motivo.
  • Stock actual: Vista del nivel de inventario de cada producto.

10. Caja

Sesión de Caja

Control de apertura y cierre de caja:

  • Abrir caja: Registrar monto inicial al comenzar el turno.
  • Cerrar caja: El sistema calcula el saldo esperado vs. el real.
  • Historial de sesiones: Registro de aperturas y cierres anteriores.
  • No se pueden realizar ventas sin una sesión de caja abierta.

Corte de Caja

Al cerrar la sesión, el sistema muestra:

  • Monto inicial.
  • Total de ventas realizadas.
  • Total de gastos registrados.
  • Saldo esperado en caja.
  • Monto real en caja (ingresado por el usuario).
  • Diferencia (sobrante o faltante).

11. Reportes

Sección de análisis con múltiples vistas:

Resumen Diario

  • Ventas del día: total, cantidad de transacciones, ticket promedio.
  • Productos más vendidos del día.
  • Métodos de pago utilizados.
  • Gráfico de barras con ventas de los últimos 7 días.

Productos

  • Productos más vendidos (por período).
  • Productos con mayor margen de ganancia.

Inventario

  • Valor total del inventario.
  • Productos con stock bajo.
  • Productos por vencer.

Gastos

  • Gastos por período.
  • Gastos por categoría.

12. Devoluciones

Gestión de devoluciones de productos:

  • Buscar venta por folio para realizar devolución.
  • Seleccionar producto(s) a devolver.
  • Indicar motivo de la devolución.
  • El sistema revierte el inventario automáticamente.
  • Historial de devoluciones realizadas.

13. Gastos

Registro de gastos operativos:

  • Descripción, monto, categoría, método de pago, fecha.
  • Categorías: Servicios, Renta, Sueldos, Impuestos, Publicidad, Otros.
  • Crear, editar y eliminar gastos.
  • Los gastos se reflejan en el corte de caja.

14. Pacientes

Base de datos de pacientes (para farmacias que manejan recetas):

  • Nombre completo, fecha de nacimiento, edad, teléfono, email.
  • Tipo de sangre, alergias, observaciones.
  • Crear, editar y buscar pacientes.
  • Ver recetas asociadas al paciente.

15. Recetas Médicas

Control de recetas:

  • Registro de receta con datos del médico (nombre, cédula profesional).
  • Asociación a un paciente.
  • Lista de medicamentos prescritos con dosis e indicaciones.
  • Fecha de emisión y fecha de vencimiento de la receta.
  • Estado: Activa, Usada, Vencida.
  • Alertas cuando una receta está próxima a vencer.

16. Alertas

Panel de notificaciones automáticas:

  • Stock bajo: Productos por debajo del mínimo configurado.
  • Productos por vencer: Próximos a caducar (30 días).
  • Recetas por vencer: Recetas próximas a expirar.
  • Marcar como leída (individual o todas).
  • Las alertas se generan automáticamente al iniciar sesión.

17. Empresa

Configuración de la empresa:

  • Nombre, RFC, teléfono, email, dirección.
  • Logo de la empresa.
  • Régimen fiscal.
  • Esta información se usa en facturas y reportes.

18. Usuarios

Administración de usuarios del sistema (solo administradores):

  • Nombre de usuario, nombre completo, email, rol.
  • Roles: Administrador, Cajero, Almacén, Consultor.
  • Crear, editar, activar/desactivar usuarios.
  • Restablecer contraseña.
  • Cada usuario tiene su propia sesión y registro de actividad.

19. Facturación CFDI

Módulo de facturación electrónica:

  • Generar factura (CFDI) a partir de una venta.
  • Datos del cliente para factura (RFC, razón social, domicilio fiscal).
  • Uso de CFDI (G01 - Adquisición de mercancías, etc.).
  • Forma de pago, método de pago.
  • Historial de facturas emitidas.
  • Vista previa y descarga (PDF/XML).
Importante: Para la emisión real de CFDI se requiere configurar el certificado digital (CSD) y la clave privada en el módulo de Empresa, así como tener conexión con el SAT.
